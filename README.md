## Реализованные пункты
1. Минимальный требуемый функционал
2. Микросервисная архитектура
3. Общение через RabbitMQ
4. Наличие docker compose
## О реализации
1. База данных - **PostgreSQL**
2. **.Net 8**
3. Сервисы выполнены по шаблону Web API, не имеют интерфейса, могут быть протестированы через **Swagger**
4. Каждый сервис имеет архитектуру **Verticle Slice Architecture**, что позволяет объединить логику и модели в рамках необходимой фичи. Также плюсом данной архитектуры, по моему мнению, является возможность гибко использовать git, так как можно сделать ветку feature-
   и работать в ней, не затрагивая другие участки кода, так как на реализацию фичи потребуется только одна папка или файл, в зависимости от стратегии реализации
6. Общение через ивенты/сообщения помогает без особых трудностей добавить дополнительные сервисы, такие как **История** и **Сервис уведомления пользователей**
7. Использование **MediatR** поможет сохранять контроллеры "тонкими", а также настроить пайпланы, например для валидации входных данных перед началом работы обработчика
8. Использование **FluentValidation** поможет легко и быстро создавать валидаторы для входных данных, моделей и т.д.
## Как использовать
### Запуск
1. Склонировать репозиторий
2. Выполнить команду `docker compose up -d`

Все миграции должны примениться автоматически      
### Адреса
Так как в данной реализации нет API шлюза, то доступ к каждому сервису осуществляется напрямую
1. #### Сервис пользователей
   http://localhost:5000
3. #### Сервис управления задачачи
   http://localhost:5001
4. #### Сервис управления проектами
   http://localhost:5002
5. #### Сервис управления группами задач
   http://localhost:5003

### P.S.
Я думал объединить группы задач, проекты и задачи в один сервис, но решил сделать их разными сервисами для демонстрации работы общения путем сообщений/ивентов<br><br>
Валидация и обработка ошибок представлены в двух сервисах - **Сервис управления задачами** **Сервис пользователей**. Так как моей целью было показать свои навыки и никого не задерживать
